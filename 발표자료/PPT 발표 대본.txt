세번째 마지막은 구성도 설계에 대한 설명입니다 
세반째는 테스트 입니다


첫번째 기술 스택 입니다

프론트 엔드 기능에는 Html, css, js 자바스크립트의 기술들을 사용합니다 그리고

백엔드 기능에는 aws 프로그램을 이용한 기능들에서 람다와 API 게이트웨이와 다이나믹DB데이터베이스와

S3 버킷와 sns와 lAM 롤 규칙와 클라우드프론트들의 기능들을 이용하여 사용합니다

그리고 테스트 및 코드 툴 기술에는 포스트맨과 비주얼 스튜디오 코드를 이용하여 사용합니다


다음은 두번째 API 설계 입니다
 
API 설계 구성도와  서버리스 아키텍처 Architecture 동작 방식 구성도 설계에 대한 설명 소개 하겠습니다

첫번째는 1. 클라이언트 Client의 이벤트 발생한다

두번째는 클라우드 프론트 기능을 이용해서 S3 버킷연결하여 S3 버킷에 HTML와 CSS와 JS 자바스크립트를 올린 파일을 연결 해줍니다 

세번째는 2. 이벤트의 엔드 포인트인 API Gateway게이트웨이로 request 들어온다

네번째는 3. API Gateway게이트웨이는 레스트 REST API 구현되어 있다 그리고 (리퀘스트 Request를 어떻게 보낼지 
get겟 /post포스트 메소드를 구현한다)

다섯번째는 람다함수 GET과 람다 함수 POST포스트는 다이나믹DB데이터베이스에 만들어둔 데이터들과 연결합니다

여섯번째는 람다함수 Post과 람다함수 get과 람다함수 make에 각각 IAM에 Role 규칙을 람다함수당 추가해서 만들어서 IAM에 Role 규칙을 연결합니다

일곱번째는 람다함수 POST 포스트는 SNS에서 만들어둔 설정을 즉 주제와 연결합니다 

여덟번째는 그리고 SNS 만들어둔 설정을 즉 주제를 람다함수MAKE도 연결합니다

아홉번째는 람다함수메이크를 S3버킷과 연결해줍니다

열번째는 4. 해당 메소드에 따라서 리퀘스트 request를 람다로 전달한다 --> 즉, 람다의 트리거 발생한다

열한번째는 5. 람다는 api gateway게이트웨이를 트리거로 가지고 있다, 이때 람다 핸들러 실행한다
람다에서 데이터를 받고, 데이터를 가공 (Response 리스폰스 를 어떻게 보낼지 람다 함수를 lambda function 구현) 합니다

열두번째는 6. 람다가 가공한 데이터 Response를 API Gateway게이트웨이로 보낸다

열세번째는 7. API Gateway게이트웨이가 받은 response를 다시 클라이언트한테 전달한다 입니다

그리고 만든는 과정 화면 중에 다이나믹DB데이터베이스 기능 화면 입니다

다음은 API 게이트웨이 구축 화면과 구축 후에 기능 화면 입니다 


목록 세번째는 테스트입니다 구축 과정과 그에 따른 테스트 결과 소개 보여드리겠습니다

만든 API 게이트웨이 스테이지 제이슨 배포 파일을 내려받고 포스트맨 협업 툴에 임포트 해서 가져오는 
화면과 가져오고 난뒤 잘 실행되는지 테스트 과정과 결과 화면 입니다

다음은 앞에서 말한 모든 기능을 이용하고 연결한 S3 버킷을 기능을 이용해서 연결하여 웹 사이트 호스팅
테스트 결과물입니다

그리고 히스토리에 가보면 저장된 데이터들과 만약 새로 입력 하면 저장 한 값 또한 볼 수 있습니다

만든 데이터 기능을 소개하자면 책 이름 book_id과 책 작가 이름 author_name과 책 장르 genre
과책 작가 폰 번호 phone_number과 출판사 publisher 기능들이 있습니다
